# Project Documentation Rules (Non-Obvious Only)

- `gui_app.py` is the main entry point for the dashboard, not `worker_main.py` (which is a CLI worker)
- `adaptive_manager.py` and `adaptive_worker.py` are the coordinated pair — manager spawns workers and handles scaling
- `worker_main.py` is the standalone worker used by `run_all_workers.py` for batch mode — it does NOT use the adaptive manager
- `gui_db.py` is read-only for the GUI layer except for `set_system_phase()` and task reset functions
- `timer_utils.py` contains `GlobalTimer` class used by both workers and manager — GUI imports `gui_db` instead of `timer_utils` for state reads
- Config JSON files in `config/` are generated by `generate_and_distribute_configs.py` — 100 worker configs with distributed city lists
- `system_state` table is the coordination mechanism between GUI, manager, and workers — single row with id=1
